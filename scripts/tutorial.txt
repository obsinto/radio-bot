# Radio BOT - Comandos dos Agentes Windows

Use o agente normal no Windows 10/11. Use o agente legado somente no Windows 7.

## Reiniciar Rapidamente

### Windows 10/11

```powershell
Stop-ScheduledTask -TaskName RadioBOTAgent
Start-ScheduledTask -TaskName RadioBOTAgent
```

### Windows 7

```powershell
schtasks.exe /End /TN RadioBOTLegacyAgent
schtasks.exe /Run /TN RadioBOTLegacyAgent
```

# Windows 10/11

O agente normal usa Node.js, Playwright, Chromium persistente e WebSocket.

## Pre-requisitos

- Windows 10 ou 11.
- Node.js LTS.
- Projeto completo do Radio BOT.
- Computador cadastrado com `DEVICE_ID` e `DEVICE_TOKEN`.
- URL WebSocket da API, por exemplo `wss://radio-api.agilytech.com/agent`.

## Instalar ou Atualizar

Execute dentro da pasta completa do projeto:

```powershell
cd C:\RadioBOTInstaller\Radio-BOT
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\windows\install-agent.ps1
```

O comando abaixo e equivalente quando o projeto esta em
`C:\RadioBOTInstaller\Radio-BOT`:

```powershell
cd C:\RadioBOTInstaller
Set-ExecutionPolicy -Scope Process Bypass
.\Radio-BOT\scripts\windows\install-agent.ps1
```

Tambem pode ser executado diretamente quando o PowerShell ja esta na raiz do
projeto:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\windows\install-agent.ps1
```

Padroes:

- Instalacao: `C:\RadioBOT`
- Tarefa: `RadioBOTAgent`
- Perfil Chromium: `C:\RadioBOT\browser-profile`
- Log: `C:\RadioBOT\logs\agent.log`

Para atualizar ou trocar o token, execute novamente o instalador. Ele copia o
projeto, recria o `.env`, substitui a tarefa e inicia o agente.

## Validar

Ver estado da tarefa:

```powershell
Get-ScheduledTask -TaskName RadioBOTAgent | Format-List TaskName,State
```

Ver ultima execucao, proxima execucao e resultado:

```powershell
Get-ScheduledTaskInfo -TaskName RadioBOTAgent | Format-List *
```

Iniciar manualmente:

```powershell
Start-ScheduledTask -TaskName RadioBOTAgent
```

Acompanhar o log:

```powershell
Get-Content C:\RadioBOT\logs\agent.log -Tail 100 -Wait
```

O log deve mostrar:

```text
[agent] conectado como <device-id>
```

## Reiniciar

```powershell
Stop-ScheduledTask -TaskName RadioBOTAgent
Start-ScheduledTask -TaskName RadioBOTAgent
```

## Desinstalar

```powershell
.\scripts\windows\uninstall-agent.ps1
```

Para remover tambem os arquivos:

```powershell
.\scripts\windows\uninstall-agent.ps1 -RemoveFiles
```

# Windows 7

O agente legado nao usa Node.js, npm ou Playwright. Ele faz polling HTTP e
controla o Chrome pela porta DevTools local. Se isso falhar, usa teclas
multimidia.

O Chrome e aberto com:

```text
--remote-debugging-port=9222
--user-data-dir=C:\RadioBOTLegacy\chrome-profile
```

## Pre-requisitos

- Windows 7 SP1.
- PowerShell 3 ou superior; recomendado WMF 5.1.
- .NET Framework atualizado e TLS 1.2 habilitado.
- Chrome instalado.
- Computador cadastrado com `DEVICE_ID` e `DEVICE_TOKEN`.
- API HTTP/HTTPS acessivel pela maquina.

## Instalar ou Atualizar

Copie esta pasta:

```text
scripts\windows7\
```

Estrutura sugerida:

```text
C:\RadioBOTInstaller\scripts\windows7\
```

Execute:

```powershell
cd C:\RadioBOTInstaller
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\windows7\install-agent.ps1
```

Padroes:

- Instalacao: `C:\RadioBOTLegacy`
- Tarefa: `RadioBOTLegacyAgent`
- Log: `C:\RadioBOTLegacy\agent.log`
- Perfil Chrome: `C:\RadioBOTLegacy\chrome-profile`
- Porta DevTools: `9222`
- Polling: 5 segundos
- `SHUTDOWN_DRY_RUN=false`

Respostas recomendadas:

```text
Pasta de instalacao: C:\RadioBOTLegacy
URL da API: https://api.seu-dominio.com
Device ID do computador: cole o DEVICE_ID
Device token: cole o DEVICE_TOKEN
Caminho do Chrome: deixe vazio para detectar automaticamente ou informe o chrome.exe
Nome da tarefa agendada: RadioBOTLegacyAgent
Simular desligamento: sim
Porta local do Chrome DevTools: 9222
Intervalo de polling: 5
```

Exemplos de caminho do Chrome:

```text
C:\Program Files\Google\Chrome\Application\chrome.exe
C:\Program Files (x86)\Google\Chrome\Application\chrome.exe
```

Para atualizar, copie novamente `scripts\windows7\` e execute o instalador. Ele
para a tarefa antiga, atualiza `run-agent.ps1`, recria `agent.config.ps1` e
inicia o agente.

## Validar

Ver estado resumido:

```powershell
schtasks.exe /Query /TN RadioBOTLegacyAgent
```

Ver estado detalhado, ultima execucao e resultado:

```powershell
schtasks.exe /Query /TN RadioBOTLegacyAgent /V /FO LIST
```

Iniciar manualmente:

```powershell
schtasks.exe /Run /TN RadioBOTLegacyAgent
```

Acompanhar o log:

```powershell
Get-Content C:\RadioBOTLegacy\agent.log -Tail 100 -Wait
```

No painel, teste nesta ordem:

1. `Estado`
2. `Captura de tela`
3. `Abrir e tocar`
4. `Play`
5. `Stop`

## Reiniciar

```powershell
schtasks.exe /End /TN RadioBOTLegacyAgent
schtasks.exe /Run /TN RadioBOTLegacyAgent
```

## Diagnosticar Autoplay

```powershell
cd C:\RadioBOTInstaller/scripts/windows7
Set-ExecutionPolicy -Scope Process Bypass
.\diagnose-autoplay.ps1
```

Para acompanhar o problema enquanto ele ocorre:

```powershell
Get-Content C:\RadioBOTLegacy\agent.log -Tail 80 -Wait
```

Confirme o caminho do `chrome.exe`, mantenha a porta `9222` e feche instancias
antigas do Chrome antes de reiniciar a tarefa.

## Desinstalar

```powershell
.\scripts\windows7\uninstall-agent.ps1
```

Para remover tambem `C:\RadioBOTLegacy`:

```powershell
.\scripts\windows7\uninstall-agent.ps1 -RemoveFiles
```

## Referencia de Configuracao

Use a URL da API, nao a URL do painel:

```text
https://radio-api.agilytech.com
```

```text
DEVICE_ID=seu-device-id
DEVICE_TOKEN=seu-device-token
```

O agente legado tambem aceita uma URL como
`wss://api.seu-dominio.com/agent` e converte para HTTPS.

## Limitacoes do Windows 7

- Nao preenche login automaticamente.
- A captura de tela inclui a area de trabalho inteira.
- O controle da pagina depende do Chrome DevTools em `127.0.0.1:9222`.
- Use `SHUTDOWN_DRY_RUN=true` apenas para testar sem desligar. Em producao,
  mantenha `false` para o botao `Desligar` e o scheduler funcionarem de fato.
